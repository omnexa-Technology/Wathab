# Frontend Code Review Skill  

**Goal:** Ensure React/Next.js (JSX) code follows project conventions, accessibility (A11y), performance, and SEO guidelines.  

**Instructions:**  
- فحص استخدام **Tailwind CSS** للتأكد من استخدام الفئات المناسبة والتنسيق الصحيح.  
- التحقق من الممارسات الأمثل في **Next.js 16** (مثلاً استخدام `Image` للصور، والتوجيه في المسارات).  
- مراجعة النصوص للتوافق مع قواعد **SEO** (مثل وجود `alt` للصور، عناوين HTML منظمة).  
- التحقق من قابلية الوصول (A11y) مثل الوسوم الدلالية (`<header>`, `<nav>`, إلخ).  
- مراجعة إدارة الحالة (**Zustand**) بحثًا عن أخطاء شائعة (مثلاً: تحديث الحالة بشكل غير مباشر).  
- فحص تكامل **GSAP** للرسوم المتحركة، والتأكد من إنهاء الـ timelines بشكل صحيح.  
- اقتراح تحسينات على الأداء (Lazy Loading، تقسيم الكود، إلخ).  

**Examples:**  
- إذا طُلب مراجعة ملف `Header.jsx`، نذكر مثلاً: “استخدم `<Image>` من Next.js بدلاً من `<img>`، وأضف نصوص بديلة لـSEO”، وهكذا.
